# MindShelf 需求分析

## 1. 项目概述

### 1.1 产品定位

MindShelf 是一款 AI 驱动的个人知识库应用，帮助用户收集、整理、检索和运用知识。用户可通过自然语言与 AI 对话，由 AI 协助管理知识库、笔记，并创建可自定义的功能页面。

### 1.2 系统组成

| 组件 | 说明 |
|------|------|
| 客户端 | Android 应用，面向终端用户；支持离线编辑，云同步为可选能力 |
| 服务端 | Flask 应用，提供 API、AI 编排、可选云同步与 SQLite 持久化 |

### 1.3 目标用户

- 需要长期积累与检索个人知识的用户
- 希望用 AI 辅助整理笔记、构建知识体系的用户
- 需要轻量自定义工具（如待办、看板）的用户

## 2. 功能需求

### 2.1 AI 对话

| 编号 | 需求 | 说明 |
|------|------|------|
| AI-01 | 多轮对话 | 支持与 AI 进行上下文连贯的多轮对话 |
| AI-02 | 会话管理 | 可创建、切换、删除对话会话 |
| AI-03 | 流式输出 | 回复以流式方式展示，提升交互体验 |
| AI-04 | 对话分支 | 参考 DeepSeek、ChatGPT：从任意消息节点可「重新提问」，产生新分支 |
| AI-05 | 分支切换 | 同一会话下可快速查看、切换不同对话分支 |
| AI-06 | 分支上下文 | 每条分支独立维护上下文；切换分支时加载对应历史 |
| AI-07 | 语音输入 | 支持语音转文字输入问题，可替代或辅助键盘输入 |
| AI-08 | 语音回答 | 支持将 AI 文字回复转为语音播放（TTS） |

**分支交互说明：**

- 用户对某条历史消息选择「重新提问」时，以该消息的上一条为上下文起点创建新分支
- 分支以树状结构组织，UI 需支持在分支间快速切换（如侧边列表、分支指示器）
- 删除会话时一并清理其下所有分支

**语音交互说明：**

- 语音输入：按住或点击麦克风录音，识别结果为可编辑文本后发送（识别引擎在架构阶段选型）
- 语音回答：AI 回复完成后可一键播放；支持暂停、停止；是否自动朗读可设置
- 需申请麦克风等系统权限，并在无网络时明确语音能力受限范围（若识别/合成依赖云端）

### 2.2 AI 服务配置

| 编号 | 需求 | 说明 |
|------|------|------|
| AI-CFG-01 | 内置服务 | 通过调用外部 API 实现；Base URL、API Key、模型名等由**服务端配置文件**统一管理 |
| AI-CFG-02 | 自定义 API | 用户可自行添加第三方 API（Base URL、API Key、模型名等）；**仅保存在本机**，不上传云端/服务端 |
| AI-CFG-03 | 提供方切换 | 对话时可选择使用内置服务（经服务端）或本机已添加的自定义 API（客户端直连） |
| AI-CFG-04 | 密钥安全 | 自定义 API Key **本机加密离线存储**，不上云；内置 API Key 仅存于服务端配置，不下发客户端 |
| AI-CFG-05 | 不同步 | 启用云同步时，自定义 API 配置**不参与同步**；换设备需用户重新在本机添加 |

**配置说明：**

- **内置 AI**：请求经 Flask 服务端转发，Key 由服务端配置文件管理
- **自定义 API**：配置存于客户端本地（加密）；对话时由客户端直接调用第三方 API，服务端不接触 Key

### 2.3 AI 联网与网页访问

| 编号 | 需求 | 说明 |
|------|------|------|
| NET-01 | 联网搜索 | AI 可搜索外部资料以回答问题 |
| NET-02 | 网页访问 | AI 可读取指定网页内容并纳入回答 |
| NET-03 | 来源标注 | 引用外部资料时标注来源链接 |

### 2.4 AI 工具调用

AI 可通过工具扩展能力，对用户数据进行读写操作。

| 编号 | 工具类型 | 说明 |
|------|----------|------|
| TOOL-01 | 知识库查询 | 搜索、列出知识库及其中内容 |
| TOOL-02 | 知识库修改 | 创建、更新、删除知识库 |
| TOOL-03 | 笔记查询 | 搜索、读取笔记 |
| TOOL-04 | 笔记修改 | 创建、更新、删除笔记 |
| TOOL-05 | 页面操作 | 创建、修改自定义页面（见 2.7） |

**约束：** 涉及写操作时需明确用户意图；敏感操作应可确认或可追溯。工具在 **Android 客户端** 执行（内置与自定义 API 共用），云同步由 `SyncCoordinator` 推送。

### 2.5 知识库

| 编号 | 需求 | 说明 |
|------|------|------|
| KB-01 | 创建知识库 | 用户可创建多个独立知识库 |
| KB-02 | 删除知识库 | 删除后进入回收站（见 2.9） |
| KB-03 | 管理知识库 | 重命名、描述、排序等基础管理 |
| KB-04 | 笔记关联 | 笔记可加入一个或多个知识库 |
| KB-05 | 内容检索 | 在知识库内搜索笔记与关联内容 |

### 2.6 笔记

| 编号 | 需求 | 说明 |
|------|------|------|
| NOTE-01 | 创建笔记 | 支持标题、正文及基础格式 |
| NOTE-02 | 编辑笔记 | 修改标题与内容 |
| NOTE-03 | 删除笔记 | 删除后进入回收站 |
| NOTE-04 | 加入知识库 | 笔记可关联到知识库 |
| NOTE-05 | 历史版本 | 保存编辑历史，支持查看与回滚（见 2.10） |

### 2.7 自定义页面

| 编号 | 需求 | 说明 |
|------|------|------|
| PAGE-01 | AI 创建页面 | 用户可通过对话让 AI 生成页面（如待办、清单、看板） |
| PAGE-02 | AI 修改页面 | 用户可通过对话调整页面布局与逻辑 |
| PAGE-03 | 数据来源 | 页面数据来自笔记和/或数据库 |
| PAGE-04 | 分享 | 页面可生成分享链接或导出 |
| PAGE-05 | 底栏固定 | 页面可添加至应用底栏，快速访问 |

**示例场景：** 用户说「帮我做一个待办页面」，AI 生成页面结构；待办项可绑定笔记或独立存储。

**范围说明：** 本节仅描述功能需求（做什么）。客户端如何渲染页面（WebView、Compose 等）属于架构/实现选型，**需求分析阶段不涉及**，留待架构设计阶段再定。

### 2.8 分享

| 编号 | 需求 | 说明 |
|------|------|------|
| SHARE-01 | 公开链接 | 分享生成公开链接，持有链接即可访问（无需登录） |
| SHARE-02 | 笔记分享 | 笔记生成公开只读链接 |
| SHARE-03 | 知识库分享 | 知识库或部分内容生成公开链接 |
| SHARE-04 | 页面分享 | 自定义页面生成公开链接 |

**约束：** 公开链接仅提供只读访问；创建者可随时撤销链接。

### 2.9 回收站

| 编号 | 需求 | 说明 |
|------|------|------|
| TRASH-01 | 软删除 | 删除的笔记、知识库、页面进入回收站 |
| TRASH-02 | 恢复 | 可从回收站恢复 |
| TRASH-03 | 永久删除 | 用户可手动永久删除；超过 **30 天**未恢复的项目自动清理 |
| TRASH-04 | 保留期限 | 回收站条目自删除之日起保留 30 天，到期后不可恢复 |

### 2.10 笔记历史版本

| 编号 | 需求 | 说明 |
|------|------|------|
| VER-01 | 自动保存版本 | 编辑时自动记录版本快照 |
| VER-02 | 版本列表 | 按时间查看历史版本 |
| VER-03 | 版本对比 | 查看两个版本差异（可分阶段实现） |
| VER-04 | 版本回滚 | 恢复到指定历史版本 |
| VER-05 | 版本上限 | 每篇笔记最多保留 **10** 个历史版本；超出时删除最旧版本 |

### 2.11 用户认证

| 编号 | 需求 | 说明 |
|------|------|------|
| AUTH-01 | 邮箱注册 | 使用邮箱完成注册 |
| AUTH-02 | 密码登录 | 支持「账号或邮箱 + 密码」登录 |
| AUTH-03 | 验证码登录 | 支持「邮箱 + 验证码」登录/注册 |
| AUTH-04 | 账号标识 | 用户可设置独立账号名，也可直接使用邮箱作为登录标识 |
| AUTH-05 | 验证码发送 | 通过 **SMTP** 发送；SMTP 参数（主机、端口、账号、授权码等）由**服务端配置文件**管理 |
| AUTH-06 | 邮件服务 | 当前使用 **QQ 邮箱** SMTP（`smtp.qq.com`）；后续可在配置中更换 |

### 2.12 离线编辑与云同步

| 编号 | 需求 | 说明 |
|------|------|------|
| SYNC-01 | 离线编辑 | 客户端默认支持离线创建、修改笔记等本地操作 |
| SYNC-02 | 云同步可选 | 云同步为可选服务；用户可仅使用本地模式 |
| SYNC-03 | 冲突处理 | 启用云同步时，采用**三路合并**：以最近一次同步的共同版本为基准，自动合并本地与云端差异；同一字段冲突时提示用户选择 |
| SYNC-04 | 登录后同步 | 用户登录服务端后，可将本地数据与云端双向同步 |
| SYNC-05 | 敏感数据排除 | 自定义 API 配置（含 Key）**不同步至云端**，仅保留在本机 |

## 4. 系统架构（概要）

```
┌─────────────────────────┐         ┌─────────────────────────┐
│  Android 客户端          │ ◄─────► │  Flask 服务端            │
│  - UI / 导航             │  HTTPS  │  - REST / WebSocket     │
│  - 本地 SQLite           │         │  - 内置 AI 转发           │
│  - 离线编辑              │         │  - LLM / 搜索代理 / 云同步 │
│  - 自定义 API（本机加密） │         └────────────┬────────────┘
└───────────┬─────────────┘                      │
            │  HTTPS（直连，Key 不出本机）           │
            ▼                                      ▼
    ┌──────────────┐                      ┌──────────┐
    │ 第三方 AI API │                      │ SQLite   │
    │（用户自定义） │                      │ (服务端)  │
    └──────────────┘                      └──────────┘
```

### 4.1 技术栈（已确定）

| 层级 | 技术 |
|------|------|
| 客户端 | Android (Kotlin) |
| 服务端 | Flask (Python) |
| 数据库 | SQLite（服务端主库；客户端本地库用于离线） |
| AI | 内置 API（服务端配置） + 本机自定义 OpenAI 兼容 API（不上云） |
| 邮件 | SMTP（配置文件）；当前 QQ 邮箱 |

### 4.2 客户端职责

- 用户界面与交互
- 本地 SQLite 存储，支持离线编辑
- 本机加密存储自定义 API 配置；客户端直连第三方 API
- AI 工具循环与写操作确认（`ToolLoopEngine` / `ClientToolDispatcher`，读写本地 Room）
- 可选：登录后与 Flask 服务端同步（不含自定义 API 配置）
- AI 对话 UI（含分支切换）

### 4.3 服务端职责

- 邮箱注册/登录（密码或验证码）
- 知识库、笔记、页面、版本、对话分支等业务 CRUD
- AI LLM 代理（`POST /ai/completions`）与联网搜索代理（`POST /ai/search`）
- 公开分享链接生成与撤销
- 读取配置文件：内置 AI、SMTP 等（见 4.4）

### 4.4 服务端配置

服务端通过配置文件（如 `config.yaml` 或 `.env`，实现阶段确定）管理以下项，**敏感信息不得提交至版本库**：

| 配置项 | 说明 | 当前值/示例 |
|--------|------|-------------|
| `ai.base_url` | 内置 AI API 地址 | 按实际 API 填写 |
| `ai.api_key` | 内置 AI API Key | 环境变量或本地配置 |
| `ai.model` | 默认模型名 | 按实际 API 填写 |
| `smtp.host` | SMTP 服务器 | `smtp.qq.com` |
| `smtp.port` | SMTP 端口 | `465`（SSL）或 `587`（TLS） |
| `smtp.user` | 发件邮箱 | QQ 邮箱地址 |
| `smtp.password` | 授权码 | QQ 邮箱 SMTP 授权码 |

*提供 `config.example.yaml` 作为模板，真实密钥由部署者本地填写。*

## 5. 核心数据实体（草案）

| 实体 | 主要字段 | 关系 |
|------|----------|------|
| User | id, email, username, passwordHash | 拥有知识库、笔记、页面 |
| KnowledgeBase | id, name, description | 包含多篇笔记 |
| Note | id, title, content, updatedAt | 可属于多个知识库；版本上限 10 |
| NoteVersion | id, noteId, content, createdAt | 属于 Note |
| CustomPage | id, name, schema, dataSource | 可固定到底栏 |
| Conversation | id, title, createdAt | AI 对话会话 |
| Message | id, conversationId, branchId, role, content, parentId | 消息节点，parentId 支持分支树 |
| Branch | id, conversationId, rootMessageId, label | 对话分支 |
| AiProvider | id, name, baseUrl, apiKey, model | **仅客户端本地**；加密存储，不上云、不同步 |
| ShareLink | id, resourceType, resourceId, token, revoked | 公开分享链接 |

*具体字段与表结构在架构设计阶段细化。*

## 6. 实施优先级

### Phase 1 — 基础能力（MVP）

- 邮箱注册/登录（密码 + 验证码）
- 笔记 CRUD、本地离线存储
- 知识库 CRUD 与笔记关联
- AI 基础对话（流式、对话分支）
- Flask 服务端 + SQLite 骨架

### Phase 2 — AI 增强

- 内置 AI + 用户自定义 API 配置
- AI 工具调用（读写知识库与笔记）
- 联网搜索与网页访问
- 语音输入、语音回答

### Phase 3 — 协作与恢复

- 可选云同步
- 回收站
- 笔记历史版本（最多 10 版）
- 公开链接分享

### Phase 4 — 自定义页面

- AI 通过 `search_custom_pages` 读取、`mutate_custom_page` 创建/修改页面 schema 与 data_bindings
- 用户编辑：默认阅读模式；TopBar「编辑」进入编辑（文本/待办/表格）；待办勾选阅读模式下可用；NoteEmbed 点击跳转笔记编辑
- 内容统一存于 `data_bindings`（含 `kind: text`）；`schema_json` 仅描述布局
- **底栏策略（dynamic_pinned）**：默认 4 Tab；存在 `pinned=true` 页面时插入第 5 Tab（页面名），最多 1 个 pinned
- 未固定页面入口：「我的」→「自定义页面」列表
- 公开分享只读页（`resource_type=page`）
- 云同步 LWW on `updated_at`；回收站 30 天保留

## 7. 已确认决策

| 事项 | 决策 |
|------|------|
| 服务端 | Flask + SQLite |
| AI 服务 | 内置 API（服务端）+ 本机自定义 API（OpenAI 兼容，不上云） |
| 认证方式 | 邮箱注册；账号/邮箱+密码 或 邮箱+验证码 |
| 离线策略 | 支持离线编辑；云同步为可选服务 |
| 分享范围 | 公开链接（只读，可撤销） |
| 版本保留 | 每篇笔记最多 10 个历史版本 |
| 对话分支 | 支持重新提问与快速切换分支（参考 DeepSeek/GPT） |
| 同步冲突 | 三路合并；同字段冲突时用户择一 |
| 回收站 | 保留 30 天，到期自动永久删除 |
| 内置 AI | 调用外部 API，由服务端配置文件管理 |
| 自定义 API | 本机加密离线保存，客户端直连；不上云、不同步 |
| 验证码 | SMTP 发送，配置文件管理；当前使用 QQ 邮箱 |
| 语音交互 | 支持语音输入（STT）与语音回答（TTS） |
| 自定义页面渲染 | Schema 驱动 Compose 组件库（见 architecture §9） |
| 底栏 pinned | dynamic_pinned：最多 1 个 pinned 页作为第 5 Tab |
